| 1 | OG Image Format (SVG → PNG/JPG) | On-Page | `social-share.svg` is used for `og:image` and `twitter:image`. Most social platforms (LinkedIn, Facebook, WhatsApp, Twitter/X) do **not** render SVG for social previews. Must be a raster image (PNG or JPG, 1200×630 px). | ❌ Needs Fix |
| 2 | Missing `datePublished` / `dateModified` on 7 of 8 Articles | Technical | Only `12-ai-tools-in-my-tech-stack` has `publishedTime` and `modifiedTime`. The remaining 7 articles in `ARTICLE_SEO` have no dates, so the `BlogPosting` JSON-LD schema ships without `datePublished`/`dateModified`. Google strongly prefers these for article rich results. | ❌ Needs Fix |
| 3 | SPA Client-Side Rendering (No SSR/SSG) | Technical | The site is a Vite + React SPA rendering all content via JavaScript in a single `<div id="root">`. Googlebot can render JavaScript, but it is slower and less reliable than SSR/SSG. All meta tags injected via `react-helmet` may not be visible to crawlers that don't execute JS (Bing, social previews, some SEO tools). Consider pre-rendering or migrating to Next.js/Astro for static generation. | ⚠️ Risk |
| 4 | Duplicate Meta Tags (index.html vs React Helmet) | On-Page | `index.html` contains hardcoded `<title>`, `<meta description>`, OG, Twitter, and JSON-LD tags. `Index.tsx` also injects the same tags via `react-helmet`. This causes duplicate tags in the DOM. The `index.html` tags are always present even on `/insights` and `/insights/:slug` routes, creating conflicting signals. | ❌ Needs Fix |
| 5 | CSP Blocking Google Analytics & Vercel Analytics | Technical | The `<meta>` CSP in `index.html` sets `connect-src 'self'` and `script-src 'self' 'unsafe-inline'`, which blocks `https://www.googletagmanager.com`, `https://va.vercel-scripts.com`, and `https://vitals.vercel-insights.com`. The `vercel.json` CSP header is broader, but the `<meta>` tag is parsed first and takes precedence in most browsers. Analytics scripts are likely silently failing. | ❌ Needs Fix |
| 6 | Missing `hreflang` Tag | On-Page | The site targets South African English (`en-ZA` in OG locale and JSON-LD `inLanguage`) but has no `<link rel="alternate" hreflang="en-za">` tag. For a `.co.za` domain, adding `hreflang` explicitly signals regional targeting to Google. | ⚠️ Missing |
| 7 | No `BreadcrumbList` Schema on Homepage or Insights Index | Technical | `BreadcrumbList` JSON-LD exists only on individual article pages (`InsightArticle.tsx`). | 8 | Sitemap `<lastmod>` Dates all Identical | Technical | Every URL in `sitemap.xml` has `<lastmod>2026-02-20</lastmod>`. Google has stated identical `lastmod` across all URLs signals unreliability, causing it to ignore `lastmod` entirely. Each URL should have its actual last-modified date. | ❌ Needs Fix |
| 9 | No `<main>` Landmark on Homepage | On-Page | `Index.tsx` renders content inside `<div className="min-h-screen">` with no `<main>` element. `Insights.tsx` and `InsightArticle.tsx` correctly use `<main>`. WCAG and SEO best practices require a `<main>` landmark for the primary content area on every page. | ⚠️ Needs Fix |
| 10 | Missing `article:published_time` on 7 Articles | On-Page | Only one article (`12-ai-tools-in-my-tech-stack`) emits `<meta property="article:published_time">`. The other 7 articles lack it. This Open Graph property helps social platforms and search engines determine content freshness. | ❌ Needs Fix |
| 11 | Hero `<h1>` Contains Only the Author Name | Content | `HeroSection.tsx` renders `<h1>Moses Nyanzi</h1>`. While the person's name is important, the `<h1>` should ideally contain the primary keyword target (e.g., "Actuarial Consultant South Africa — Moses Nyanzi") to match the `<title>` tag. | ⚠️ Optimise |
| 12 | Internal Links Use `<a href>` Instead of React Router | Technical | All internal navigation (`/`, `/insights`, `/insights/:slug`) uses plain `<a href="...">` tags, causing full-page reloads in the SPA. While not directly an SEO issue, full reloads on a JS-rendered site mean Googlebot must re-execute the full JS bundle for every internal crawl. Use React Router `<Link>` for client-side navigation. | ⚠️ Optimise |
| 13 | Missing `tel:` or Structured Contact Info | Off-Page | The `ProfessionalService` JSON-LD includes `email` but no `telephone`. For local SEO, Google prefers a phone number in structured data to display in knowledge panels. | ⚠️ Missing |
